# 帖子删除接口对接规范

用于「图文详情 / 视频详情 / 文章详情」页面：当帖子的作者是当前登录用户时，展示删除按钮，删除需二次确认。

> 前端已按该规范实现：点击删除 -> 弹窗确认“是否确定删除？” -> 调用接口 -> 成功后返回上一页。

---

## 1. 接口定义

- **Method**：`POST`
- **Path**：`/api/app/v1/post/delete`
- **Auth**：需要登录（token）
- **权限**：仅允许删除本人帖子（作者）

---

## 2. 请求参数

### 2.1 Body（JSON）

```json
{
  "postId": 123456,
  "postType": 1
}
```

字段说明：
- `postId`：帖子 ID（`number | string`）
- `postType`：帖子类型（`1=图文 2=视频 3=文章`），建议必传（后端可用于分表/路由）

---

## 3. 返回结构

### 3.1 成功

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "postId": 123456,
    "deleted": true
  }
}
```

字段说明：
- `data.postId`：被删除的帖子 ID
- `data.deleted`：是否删除成功（建议固定 `true`）

> 说明：前端不强依赖 `data` 字段内容，只要 `code=200` 即认为成功。

### 3.2 失败（示例）

```json
{
  "code": 403,
  "msg": "forbidden",
  "data": null
}
```

---

## 4. 业务约束（建议）

- 若帖子不存在：建议返回 `404`。
- 若非作者删除：建议返回 `403`。
- 若帖子已删除：建议幂等处理（重复删除返回成功或返回特定业务码均可，但建议返回成功，避免前端误判）。
