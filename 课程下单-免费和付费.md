## 1. 获取入学信息详情

### 请求信息

- **接口地址**：`/api/course/enrollment-detail`
- **请求方式**：GET
- **接口说明**：用于课程「入学信息」页面，返回用户手机号和支付金额。

### 请求参数（Query）

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | number | 是 | 课程 ID |

### 请求示例

```http
GET /api/course/enrollment-detail?id=1001
```

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "phone": "15122443037",
    "payAmount": "1.00"
  }
}
```

### 响应字段说明

顶层字段：

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| code | number | 状态码，`200` 表示成功 |
| message | string | 响应消息 |
| data | object | 业务数据 |

`data` 字段：

| 字段名 | 类型 | 必返 | 说明 |
| --- | --- | --- | --- |
| phone | string | 否 | 手机号（用于入学信息展示） |
| payAmount | string | 否 | 支付金额；为空字符串 `""` 或不返回表示免费领取 |

### 业务约定

- `payAmount` 有值（如 `"1.00"`）表示需支付对应金额。
- `payAmount` 为空字符串或缺省，前端按“免费领取”处理。

### 错误响应示例

```json
{
  "code": 400,
  "message": "参数错误",
  "data": null
}
```

## 2. 创建入学支付订单（APP 微信支付）

### 请求信息

- **接口地址**：`/api/app/v1/course/enrollment/pay`
- **请求方式**：POST
- **接口说明**：当 `payAmount` 有值（需要支付）时调用，后端返回 APP 微信支付参数，前端用于 `uni.requestPayment` 拉起收银台。

### 请求参数（Body）

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | number | 是 | 课程 ID |
| phone | string | 是 | 用户手机号 |
| age | string | 是 | 年龄区间（如 `26-30`） |

### 请求示例

```http
POST /api/app/v1/course/enrollment/pay
Content-Type: application/json

{
  "id": 100000001,
  "phone": "15122443037",
  "age": "31-40"
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "provider": "wxpay",
    "orderId": "ENROLL_100000001_1741160000000",
    "orderInfo": {
      "appid": "wx_mock_appid",
      "noncestr": "nonce_1741160000000",
      "package": "Sign=WXPay",
      "partnerid": "1900000109",
      "prepayid": "wx201410272009395522657a690389285100",
      "timestamp": "1741160000",
      "sign": "MOCK_SIGN"
    }
  }
}
```

### 响应字段说明

顶层字段：

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| code | number | 状态码，`200` 表示成功 |
| message | string | 响应消息 |
| data | object | 业务数据 |

`data` 字段：

| 字段名 | 类型 | 必返 | 说明 |
| --- | --- | --- | --- |
| provider | string | 否 | 支付渠道，APP 微信支付通常为 `wxpay` |
| orderId | string | 否 | 后端订单号（用于查询和对账） |
| orderInfo | object / string | 是 | 透传给 `uni.requestPayment` 的订单参数 |

`orderInfo`（object 形式）常用字段：

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| appid | string | 微信开放平台应用 ID |
| noncestr | string | 随机串 |
| package | string | 固定值常见为 `Sign=WXPay` |
| partnerid | string | 微信商户号 |
| prepayid | string | 预支付交易会话标识 |
| timestamp | string | 时间戳（秒） |
| sign | string | 签名 |

## 3. 创建免费领取课程

### 请求信息

- **接口地址**：`/api/app/v1/course/claim`
- **请求方式**：POST
- **接口说明**：当 `payAmount` 为空或者为0（免费课程）时调用。

### 请求参数（Body）

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | number | 是 | 课程 ID |
| phone | string | 是 | 用户手机号 |
| age | string | 是 | 年龄区间（如 `26-30`） |

### 请求示例

```http
POST /api/app/v1/course/claim
Content-Type: application/json

{
  "id": 100000001,
  "phone": "15122443037",
  "age": "31-40"
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 12345,
    "courseId": 100000001,
    "orderNo": "CO20260318112233999999",
    "enrollTime": "2026-03-18 11:22:33"
  }
}
```

### 响应字段说明

顶层字段：

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| code | number | 状态码，`200` 表示成功 |
| message | string | 响应消息 |
| data | object | 业务数据 |

`data` 字段：

| 字段名 | 类型 | 必返 | 说明 |
| --- | --- | --- | --- |
| id | number | 是 | 用户课程记录 ID |
| courseId | number | 是 | 课程 ID |
| orderNo | string | 是 | 免费领取生成的订单号（`app_course_order.order_no`） |
| enrollTime | string | 是 | 领取时间，格式 `Y-m-d H:i:s` |

## 4. APP 端支付流程（前端实现）

页面：`/pages/course/enrollment?id=课程ID`

1. 页面加载时调用 `/api/course/enrollment-detail`，获取 `phone` 和 `payAmount`。
2. 用户点击提交后先校验：年龄已选、手机号格式正确。
3. 若 `payAmount` 为空：走免费领取分支（当前待后端领取接口）。
4. 若 `payAmount` 有值：仅在 APP 端继续，调用 `/api/app/v1/course/enrollment/pay` 创建订单。
5. 前端拿到 `orderInfo` 后调用 `uni.requestPayment`（`provider: 'wxpay'`）拉起微信支付。
6. `success` 回调提示“支付成功”；`fail` 按取消/失败分别提示。

简化调用示例：

```ts
const payOrder = await createEnrollmentPayOrder({
  id: courseId,
  phone,
  age,
})

await uni.requestPayment({
  provider: 'wxpay',
  orderInfo: payOrder.orderInfo,
})
```

---

## 4. 联调注意事项

- 前端只负责拉起支付，签名与预下单必须由后端完成。
- 支付结果以服务端回调/查单为准，前端 `success` 仅代表客户端支付流程完成。
- 非 APP 端（H5/小程序）当前不发起该支付流程。
- 建议后续补充“支付完成页”与“订单状态查询接口”做业务闭环。
